---
title: Admin Apps
description: Create, list, retrieve, update, delete, and rotate API tokens for ClawHost apps.
---

These endpoints manage apps — the top-level tenant unit in ClawHost. Each app has its own `api_token` used to authenticate bot API requests.

All endpoints require an admin token.

```bash
Authorization: Bearer <admin-token>
```

---

## Create app

```
POST /bot/api/v1/admin/apps
```

Creates a new app and generates a unique `api_token` for it.

### Request body

<ParamField body="name" type="string" required>
  Display name for the app.
</ParamField>

<ParamField body="url" type="string">
  URL associated with the app (optional).
</ParamField>

<ParamField body="description" type="string">
  Description of the app (optional).
</ParamField>

<ParamField body="owner_email" type="string">
  Email address of the app owner (optional).
</ParamField>

<ParamField body="bot_domain_template" type="string">
  Template for bot subdomain URLs within this app (optional). Overrides the global `domain.bot_domain_template` setting.
</ParamField>

### Response

<ResponseField name="id" type="string">
  Unique app identifier (UUID).
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the app.
</ResponseField>

<ResponseField name="api_token" type="string">
  Bearer token used to authenticate bot API requests for this app. Store this securely — it is returned in full only on creation and token reset.
</ResponseField>

<ResponseField name="url" type="string">
  URL associated with the app.
</ResponseField>

<ResponseField name="description" type="string">
  Description of the app.
</ResponseField>

<ResponseField name="owner_email" type="string">
  Email address of the app owner.
</ResponseField>

<ResponseField name="bot_domain_template" type="string">
  Bot subdomain template for this app.
</ResponseField>

<ResponseField name="status" type="string">
  App status. One of `active` or `disabled`. Defaults to `active`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the app was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last update.
</ResponseField>

### Example

```bash
curl -X POST http://<host>:18080/bot/api/v1/admin/apps \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-app"}'
```

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "name": "my-app",
    "api_token": "d4e5f6a7b8c9...",
    "url": "",
    "description": "",
    "owner_email": "",
    "bot_domain_template": "",
    "status": "active",
    "created_at": "2026-03-24T10:00:00Z",
    "updated_at": "2026-03-24T10:00:00Z"
  }
}
```

## List apps

```
GET /bot/api/v1/admin/apps
```

Returns all apps, ordered by creation time (newest first).

### Response

An array of App objects. See [Create app](#create-app) for field descriptions.

### Example

```bash
curl http://<host>:18080/bot/api/v1/admin/apps \
  -H "Authorization: Bearer <admin-token>"
```

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "my-app",
      "api_token": "d4e5f6a7b8c9...",
      "status": "active",
      "created_at": "2026-03-24T10:00:00Z",
      "updated_at": "2026-03-24T10:00:00Z"
    }
  ]
}
```

## Get app

```
GET /bot/api/v1/admin/apps/:id
```

Returns a single app by ID.

### Path parameters

<ParamField path="id" type="string" required>
  App ID (UUID).
</ParamField>

### Response

A single App object. See [Create app](#create-app) for field descriptions.

### Example

```bash
curl http://<host>:18080/bot/api/v1/admin/apps/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <admin-token>"
```

## Update app

```
PUT /bot/api/v1/admin/apps/:id
```

Updates one or more fields on an app. Only fields included in the request body are updated.

### Path parameters

<ParamField path="id" type="string" required>
  App ID (UUID).
</ParamField>

### Request body

<ParamField body="name" type="string">
  New display name for the app.
</ParamField>

<ParamField body="url" type="string">
  Updated URL.
</ParamField>

<ParamField body="description" type="string">
  Updated description.
</ParamField>

<ParamField body="owner_email" type="string">
  Updated owner email.
</ParamField>

<ParamField body="bot_domain_template" type="string">
  Updated bot domain template.
</ParamField>

<ParamField body="status" type="string">
  App status. Must be `active` or `disabled`.
</ParamField>

### Response

The updated App object. See [Create app](#create-app) for field descriptions.

### Example

```bash
curl -X PUT http://<host>:18080/bot/api/v1/admin/apps/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "renamed-app", "status": "active"}'
```

## Delete app

```
DELETE /bot/api/v1/admin/apps/:id
```

Deletes an app by ID.

<Warning>
  Deleting an app does not automatically stop or remove the bots that belong to it. Clean up running bots before deleting their app.
</Warning>

### Path parameters

<ParamField path="id" type="string" required>
  App ID (UUID).
</ParamField>

### Response

<ResponseField name="message" type="string">
  Confirmation message (`"app deleted"`).
</ResponseField>

### Example

```bash
curl -X DELETE http://<host>:18080/bot/api/v1/admin/apps/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <admin-token>"
```

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "app deleted"
  }
}
```

## Reset app API token

```
POST /bot/api/v1/admin/apps/:id/reset-token
```

Generates a new `api_token` for the app and invalidates the previous one. Any bots using the old token will immediately lose access.

### Path parameters

<ParamField path="id" type="string" required>
  App ID (UUID).
</ParamField>

### Response

<ResponseField name="api_token" type="string">
  The newly generated API token. This is the only time it is returned in full.
</ResponseField>

### Example

```bash
curl -X POST http://<host>:18080/bot/api/v1/admin/apps/a1b2c3d4-e5f6-7890-abcd-ef1234567890/reset-token \
  -H "Authorization: Bearer <admin-token>"
```

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "api_token": "f7g8h9i0j1k2..."
  }
}
```
